Eje 4: Propuesta, análisis y toma de decisiones
Panorama del proceso
¿Qué busca este eje?
Este eje muestra cómo pasar de una idea inicial a una propuesta sustentada con información, análisis, evidencias y decisiones responsables.
Idea central: no se trata solo de buscar información. Se trata de organizarla, analizarla, validarla y usarla para proponer una solución o mejora.
Los 4 pilares de la analítica de datos
⬇️
Recolección de datos
Se reúne información inicial desde bases de datos, documentos, encuestas, observaciones o fuentes externas.
🧹
Limpieza y preparación
Se revisa la calidad de la información, se organiza, se depura y se deja lista para analizar.
📊
Análisis e interpretación
Se identifican patrones, relaciones, hallazgos, causas posibles y oportunidades de mejora.
🖥️
Visualización y comunicación
Se presentan resultados mediante tablas, gráficos, informes o dashboards para facilitar decisiones.

Lluvia de ideas
¿Qué se hace?
La lluvia de ideas permite explorar diferentes problemas, necesidades u oportunidades antes de elegir un tema definitivo. Primero se listan opciones y luego se revisan.
Cómo organizar las ideas
Tipo de ideaPregunta orientadoraEjemplo general
Problema¿Qué situación necesita solución?Baja participación, pérdida de datos, poca eficiencia.
Oportunidad¿Qué se podría mejorar?Automatizar procesos, mejorar ventas, optimizar recursos.
Necesidad¿Qué hace falta?Capacitación, información confiable, indicadores o seguimiento.
Innovación¿Qué herramienta podría ayudar?Dashboard, app, IA, automatización o sistema de información.
Instrucción
Escriban mínimo cinco ideas posibles y expliquen brevemente por qué cada una podría convertirse en una propuesta de análisis o mejora.

Selección del tema
¿Cómo se elige?
Después de la lluvia de ideas, se selecciona el tema más pertinente. La elección debe basarse en criterios, no solo en gusto personal.
Criterios
CriterioPregunta de revisión
Pertinencia¿El tema responde a una necesidad real?
Viabilidad¿Se puede desarrollar con el tiempo y recursos disponibles?
Información disponible¿Existen datos o fuentes para analizarlo?
Impacto¿La propuesta puede generar mejora o decisión útil?
Claridad¿El tema puede explicarse de forma concreta?
Resultado esperado
El grupo debe tener un tema delimitado.

Búsqueda de fuentes
¿Para qué?
Las fuentes permiten sustentar la propuesta. Una idea sin evidencia queda como opinión; una idea con fuentes puede convertirse en argumento.
Tipos de fuentes
FuenteQué aportaCuidado
Datos internosInformación del contexto.Verificar calidad y consistencia.
AcadémicasConceptos y antecedentes.Revisar autor, fecha y publicación.
InstitucionalesNormas, informes o estadísticas.Confirmar que sean oficiales.
EntornoTendencias, competencia o mercado.No usar datos sin origen verificable.
IAApoyo para organizar o redactar.Validar todo antes de usarlo.

Seguridad, ética e IA responsable
Seguridad
RiesgoQué puede pasarCómo se controla
Perder archivosSe pierde el avance.Guardar copias de seguridad.
Modificar datos sin controlResultados no confiables.Usar versiones y permisos.
Usar fuentes no verificadasPropuesta mal sustentada.Registrar y validar fuentes.
Copiar sin citarPuede haber plagio.Citar correctamente.
Usar IA sin revisarErrores o datos inventados.Validar con fuentes reales.
Ética
  • No inventar datos, fuentes ni resultados.
  • No manipular información para favorecer una idea.
  • No ocultar limitaciones del análisis.
  • No presentar texto de IA como análisis propio sin revisión.
  • No usar información sensible sin autorización.
Regla final: toda decisión debe poder explicarse con datos, fuentes y criterios verificables.
