SalidaMostrar Suma

Nivel 4

División de dos números

La división introduce una validación importante: no se puede dividir entre cero. Por eso este nivel agrega una decisión dentro del algoritmo.

EntradaA y B
ProcesoSi B ≠ 0 entonces Cociente ← A / B
SalidaResultado o mensaje de error
1
Inicio.

Arranca el proceso.

2
Leer A y B.

Se ingresan dividendo y divisor.

3
Validar.

Se revisa si B es igual a 0.

4
Decidir.

Si B es distinto de 0 se divide; si no, se muestra error.

5
Fin.

Se termina el algoritmo.

💻 algoritmo_division.algo
Algoritmo DividirDosNumeros
  Definir A, B, Cociente Como Real
  Escribir "Ingrese el primer número:"
  Leer A
  Escribir "Ingrese el segundo número:"
  Leer B
  Si B <> 0 Entonces
     Cociente <- A / B
     Escribir "La división es: ", Cociente
  SiNo
     Escribir "Error: no se puede dividir entre cero"
  FinSi
FinAlgoritmo

Nivel 5

Integración de las cuatro operaciones: calculadora

La calculadora junta las cuatro operaciones en una sola estructura. La diferencia frente a los niveles anteriores es que ahora el algoritmo primero debe leer una opción y luego decidir qué operación ejecutar.

Opción 1Suma
Opción 2Resta
Opción 3Multiplicación
Opción 4División
1
Mostrar menú.

El usuario elige qué operación quiere realizar.

2
Leer opción.

Se captura un número entre 1 y 4.

3
Leer A y B.

Se capturan los dos operandos.

4
Decidir operación.

Según la opción, se suma, resta, multiplica o divide.

5
Validar división.

Si la opción es dividir, hay que revisar si B es cero.

6
Mostrar resultado.

Se presenta el valor calculado o un error.

Algoritmo CalculadoraBasica
  Definir A, B, Resultado Como Real
  Definir Opcion Como Entero

  Escribir "1. Sumar"
  Escribir "2. Restar"
  Escribir "3. Multiplicar"
  Escribir "4. Dividir"
  Leer Opcion

  Escribir "Ingrese el primer número:"
  Leer A
  Escribir "Ingrese el segundo número:"
  Leer B

  Segun Opcion Hacer
    1:
      Resultado <- A + B
      Escribir "Resultado: ", Resultado
    2:
      Resultado <- A - B
      Escribir "Resultado: ", Resultado
    3:
      Resultado <- A * B
      Escribir "Resultado: ", Resultado
    4:
      Si B <> 0 Entonces
        Resultado <- A / B
        Escribir "Resultado: ", Resultado
      SiNo
        Escribir "Error: no se puede dividir entre cero"
      FinSi
    De Otro Modo:
      Escribir "Opción no válida"
  FinSegun
FinAlgoritmo

Nivel 6

Sugerencias de algoritmos fáciles para escribir

Ejercicios básicos para practicar. La idea es escribirlos en el cuaderno, representar el DFD y el Pseudocódigo. con palabras claras usando la estructura: Inicio → Leer → Procesar → Mostrar → Fin.

1. Número par o impar

Lee un número, revisa si al dividirlo entre 2 el residuo es 0 y muestra si es par o impar.

  1. Leer N.
  2. Si N mod 2 = 0, mostrar "Par".
  3. Si no, mostrar "Impar".

2. Mayor de dos números

Lee A y B y muestra cuál es mayor. Si son iguales, también se puede indicar.

  1. Leer A y B.
  2. Si A > B, mostrar A.
  3. Si B > A, mostrar B.
  4. Si A = B, mostrar "Son iguales".

3. Promedio de tres notas

Lee tres calificaciones, calcula el promedio y muestra el resultado.

  1. Leer N1, N2 y N3.
  2. Promedio ← (N1 + N2 + N3) / 3.
  3. Mostrar Promedio.

4. Área de un rectángulo

Lee base y altura, calcula el área y la muestra.

  1. Leer Base y Altura.
  2. Área ← Base * Altura.
  3. Mostrar Área.

5. Tabla de multiplicar

Lee un número y escribe su tabla del 1 al 10. Este ya introduce repetición.

  1. Leer N.
  2. Repetir desde i = 1 hasta 10.
  3. Mostrar N * i.

6. Conversión de centímetros a metros

Lee una cantidad en centímetros, conviértela a metros y muestra el nuevo valor.

  1. Leer Cm.
  2. Metros ← Cm / 100.
  3. Mostrar Metros.

Consejos para que te salgan bien 🌸

1
Usa verbos claros.

Leer, calcular, comparar, mostrar.

2
Un paso por línea.

Así el algoritmo se entiende mejor.

3
Nombra bien las variables.

Ejemplo: nota1, nota2, promedio.

4
Valida cuando haga falta.

Como evitar dividir entre cero.

5
Empieza por lo más simple.

Primero suma, luego comparación, luego ciclos.
