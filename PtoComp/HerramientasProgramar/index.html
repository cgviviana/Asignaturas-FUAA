Conexión con el Pensamiento Computacional

  • Descomposición: sprites/escenas/eventos; pantallas/componentes.
  • Abstracción: bloques/funciones personalizadas para ocultar detalles.
  • Patrones: bucles de juego, detección de colisión, manejadores de eventos.
  • Algoritmos: secuencias, decisiones e iteraciones visibles.

Programación por Código (Python · Arduino · Kivy)

Control, escalabilidad y ecosistemas maduros; requiere dominio de sintaxis, librerías y herramientas.

Python Multipropósito Web/Escritorio

Lenguaje legible con gran ecosistema (datos, automatización, web). Excelente para enseñar algoritmia con claridad.

# Decisión + composición (FizzBuzz) for i in range(1, 21): out = ("Fizz" if i%3==0 else "") + ("Buzz" if i%5==0 else "") print(out or i)

Arduino (C/C++) Microcontroladores Hardware/Robótica

Programa microcontroladores para leer sensores y actuar sobre salidas. Útil cuando el curso incluye interacción con dispositivos físicos.

// Blink void setup(){ pinMode(LED_BUILTIN, OUTPUT); } void loop(){ digitalWrite(LED_BUILTIN, HIGH); delay(500); digitalWrite(LED_BUILTIN, LOW); delay(500); }

Python con Kivy GUI Escritorio Móvil (Android/iOS)

Framework para crear interfaces y apps multiplataforma con Python. Aporta transición de scripts a aplicaciones con UI.

from kivy.app import App from kivy.uix.label import Label class HolaApp(App): def build(self): return Label(text="Hola Kivy") if __name__ == "__main__": HolaApp().run()

Kivy.org Documentación

¿Cuándo usar cada una? (Código)

  • Web/Escritorio: Python (scripts, análisis, backends); Kivy (apps con UI).
  • Móvil: Kivy (empaquetado adicional para Android/iOS).
  • Hardware/Robótica: Arduino (cuando el curso contemple electrónica básica/IoT educativo).

Programación Mixta (mBlock)

Puente bloques ↔ código: observar la equivalencia semántica para comprender cómo se traduce la lógica visual al texto.

mBlock Transición Hardware/Robótica

Basado en Scratch, añade soporte para Arduino/micro:bit y permite alternar a C++/Python. Ideal para cursos STEAM que combinan lógica y dispositivos.

/* mBlock → Arduino (conceptual) cada 1s: luz = analogRead(A0) if luz < 300 → LED ON else → LED OFF */

mBlock · Sitio oficial Documentación

Conexión con Pensamiento Computacional

  • Descomposición: separar lectura de sensores, lógica y actuadores.
  • Abstracción: umbrales y estados como constantes/variables.
  • Patrones: ciclo lectura→decisión→acción; temporizaciones y antirrebote.
  • Algoritmos: if/else, máquinas de estado, pruebas de casos límite.
Nota: usar Mixta cuando el objetivo incluya hardware/robótica o transición guiada del bloque al texto.
