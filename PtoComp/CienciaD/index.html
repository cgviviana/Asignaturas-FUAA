De los datos a la Ciencia de Datos

Caso práctico "En la Heladería de Sammy"

¿Qué es un Dato?

Un dato es un valor crudo sin contexto. Por sí solo, no dice mucho. En nuestra heladería, los datos son:

  • 25°C (temperatura)
  • Vainilla (sabor)
  • 11:00 a.m. (hora de venta)
  • $12.500 (precio)

De Datos a Información

La información surge cuando los datos se organizan y se les da un contexto. Ahora tienen sentido.

Ejemplo:
"A las 11:00 a.m. , la temperatura era de 25°C y vendimos un helado de Vainilla por $12.500."

De Información a Conocimiento

El conocimiento es la comprensión de la información. Se obtiene al analizar patrones. Nos ayuda a entender el ¿por qué?.

Ejemplo:
"Los días que superan los 25°C, las ventas de helados aumentan un 50%."

De Conocimiento a Acción (Sabiduría)

Convertimos el patrón en reglas de decisión operativas (simples y ejecutables):

  • Si T ≥ 27°C y no llueve → producir +20%. [sube la demanda con calor]
  • Si llueve → producir -15% y priorizar domicilios. [Baja la afluencia a la tienda].
  • Si hay evento cerca → stock +30% de sabores top. [La demanda se concentra en sabores favoritos]

Sabiduría = decidir cuánto producir hoy con base en el conocimiento.

El centro del proceso: El Modelo

Un modelo de datos es una herramienta matemática [Función o formulación] que aprende de los datos históricos para predecir un resultado futuro o entender relaciones.

En nuestro caso, el modelo podría predecir las ventas diarias de helado.

Paso 1: Formular la Pregunta de Negocio

Pregunta: "¿Cuántos helados venderemos mañana para no quedarnos sin existencias?"

Esta pregunta es clave porque nos obliga a usar datos para predecir un valor (número de helados).

Paso 2: Recolectar y Organizar Datos

Integrar datos de diferentes fuentes (fecha, temperatura, lluvia sí/no, ventas, eventos), limpiar y organizarlos en una dataset para analizar.

Paso 3: Explorar los Datos (EDA)


Explorar el dataset y sus gráficos, para validar calidad y descubrir patrones.

Paso 3 : ¿Qué gráficos podemos usar?

La elección del gráfico depende de la pregunta (algunos ejemplos):

  • **Gráfico de Barras:** Para comparar categorías, cuál es mayor/menor y ordenar. Ej.: Ventas por **sabor** -> "Vainilla se vende más que Chocolate."
  • **Gráfico de Líneas:** Para ver evolución en tiempo. Ej: Ventas por día en el trimestre" o **Tendencias** -> "Las ventas alcanzan su pico en agosto."
  • **Histograma:** Para ver concentración o distribución de los números por rango Ej.: **Temperaturas.** -> "≥27 °C se define caluroso, si el pronóstico es ese rango producir más, con frío se prioriza domicilio."

Paso 4: Construir y Entrenar el Modelo

Con la pregunta ** “¿cuántos helados preparar para no quedarme sin existencias?”** y los datos históricos (temperatura, lluvia, día), se ajusta una fórmula que estima (con mínimo de error) , cuántos helados preparar. .

Ejemplo:. Ventas = (5 * Temperatura) - (10 * Lluvia) + 50 [Con lluvia =1 Sin lluvia =0]

Paso 4 : ¿Qué significan los números del modelo?

Estos números son los coeficientes del modelo. Nos dicen cuánto influye cada factor en las ventas:

  • 5: Por cada grado que sube la temperatura, el modelo predice un aumento de 5 helados en las ventas.
  • -10: Si llueve, el modelo predice una caída de 10 helados en las ventas.
  • 50: Es la base de ventas esperada en un día "promedio", sin considerar los factores anteriores.

Tipos de Modelo

La ciencia de datos ¿Qué otros problemas podría resolver?

  • Modelo de Predicción (regresión): Para planear producción → ¿Cuántos helados se venderán?.
  • Modelo de Clasificación: Para predecir qué clientes tienen mayor probabilidad de comprar un nuevo sabor → Cupón / Ofertas.
  • Modelo de Agrupación (Clustering): Para agrupar a los clientes en segmentos→ Promoción: "sabores clásicos" o "exploradores" .
  • Modelo de Recomendación: Para dar sugerencias a cada cliente→ Combos/sabores según su historial..

Paso 5: Evaluación del Modelo

Para saber si el modelo es bueno, se prueba con datos que no ha visto y se ajusta la fórmula si es necesario para que sea más exacta.

→ Si el modelo predice 100 helados y la tienda vendió 98, el modelo es preciso.

Paso 6: Aplicación en la Desición

El modelo ahora está listo para usar solo se integra al un sistema y se ingresa la temperatura prevista y el modelo da una predicción de ventas.

Según el modelo, si mañana la temperatura es de 28°C, predecimos una venta de 120 helados.

Medir el Éxito con indicadores

Un KPI (Key Performance Indicator) son los objetivos que queremos mejorar.

En la heladería, los KPIs importantes podrían ser:

  • Ventas diarias (unidades)→ Fuente: POS (caja/tickets) → Planear producción.
  • Desperdicio % →Fuente: Producción (producido) + POS (vendido) → Reducir sobrantes/costos.
  • Predicción de ventas (mañana) → Fuente: Clima + historial de ventas → Ajustar stock y personal.
  • Tiempo de espera promedio → Fuente: Operación/Caja → Asignar personal y filas.
  • Satisfacción (bueno/neutral/malo)→ Fuente: Opiniones → Mejorar sabores y servicio.

Meta: más ventas, menos desperdicio, clientes felices.

Del Dato a la decisión

Se empieza con datos crudos, que se transforman en información, conocimiento y, finalmente, en una acción estratégica "preparar más helado" que impacta directamente en el éxito de la heladería.

Se puede decir que **la ciencia de datos** es una súper herramienta que transforma la información en valor.
