Diagmando el proceso

El diagrama de flujo permitirá representar, paso a paso, cómo un sistema puede registrar datos, revisarlos, clasificarlos y generar alertas o reportes útiles.

El DFD permite pasar de una problemática real a una solución lógica que pueda convertirse en la solución , en nuestro caso el sistema de información.

Retomando

Pensamiento computacional

El pensamiento computacional ayuda a mirar un problema con lógica: dividirlo, reconocer lo que se repite, seleccionar los datos importantes y organizar una secuencia de pasos para resolverlo.

DescomponerSeparar la situación en partes manejables.
Reconocer patronesIdentificar repeticiones o casos frecuentes.
AbstraerQuedarse con los datos necesarios.
Diseñar pasosDefinir el recorrido de la solución.

Situación problema

Escuela Rural Las Lomitas

La escuela necesita organizar información que normalmente puede quedar dispersa en cuadernos, mensajes, planillas o conversaciones con las familias.

El sistema de información ayudaría a registrar datos escolares, familiares, académicos y administrativos para hacer seguimiento de manera más clara.

EstudianteNombre, grado, vereda y jornada.
AcudienteNombre, teléfono y parentesco.
AsistenciaFecha, hora de llegada y estado.
SeguimientoEstado académico y novedades.
Dato clave

Estado de cartera

Para el seguimiento administrativo, el campo puede llamarse estado de cartera o estado de pago.

Al díaNo presenta valores pendientes.
PendienteTiene un pago por revisar.
En moraPresenta retraso en el pago.
Acuerdo de pagoExiste compromiso registrado.
Datos del sistema

Qué información debería registrar

Antes de dibujar el flujo, se deben definir los datos que entran al sistema. Estos datos permiten tomar decisiones y generar reportes.

IdentificaciónNombre del estudiante, grado, vereda, jornada.
Contacto familiarAcudiente, teléfono, parentesco, canal de contacto.
AsistenciaFecha, hora de llegada, puntualidad, ausencias.
Estado académicoDesempeño, asignaturas con dificultad, observaciones.
Estado de carteraAl día, pendiente, en mora o acuerdo de pago.
Condiciones de accesoConectividad, dispositivo, transporte y necesidad principal.
Datos cualitativos

Datos que describen una situación

Los datos cualitativos no son números. Describen características, condiciones o estados del estudiante y su contexto.

VeredaLugar de procedencia del estudiante.
Motivo de ausenciaSalud, transporte, familiar, clima u otro.
Estado académicoAlto, básico, bajo o en seguimiento.
Necesidad principalConectividad, transporte, apoyo académico o familiar.
Datos cuantitativos

Datos que permiten contar o calcular

Los datos cuantitativos son numéricos. Sirven para medir, comparar y activar alertas cuando se supera un límite.

Hora de llegadaPermite calcular retardos.
Número de ausenciasPermite detectar reincidencia.
Promedio académicoPermite priorizar acompañamiento.
Días en moraPermite revisar estado de cartera.

Preguntas del flujo

Decisiones que debe tomar el sistema

Las decisiones son puntos donde el sistema necesita preguntar algo para seguir una ruta u otra.

¿Los datos están completos?Si falta información, se devuelve al registro.
¿Llegó tarde?Si la hora supera el límite, se marca retardo.
¿Tiene bajo desempeño?Si requiere apoyo, se genera seguimiento académico.
¿Tiene cartera pendiente?Si está en mora, se genera alerta administrativa.

Uso responsable

Cuidado con la información

Los datos de estudiantes y familias deben usarse con responsabilidad. No se registran para señalar o exponer a nadie, sino para acompañar, priorizar apoyos y tomar mejores decisiones.

Importante: solo recoger datos necesarios, protegerlos y usarlos con un propósito claro.
