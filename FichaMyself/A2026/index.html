Paso 1. Contextualizar la problemática

Primero se debe presentar una situación real del aula infantil. No se empieza por la herramienta, sino por una necesidad concreta del contexto educativo.

Qué debe hacer el grupo

Escribir una problemática observable en niños pequeños. Debe ser clara, cercana al aula y posible de convertir en una actividad lúdica. Ejemplo: los niños confunden plástico y papel, no reconocen emociones básicas, no diferencian formas o colores, o presentan dificultad para seguir una secuencia sencilla.

Qué debe explicar

  • Qué está pasando.
  • Por qué es importante resolverlo.
  • Qué edad o nivel de niños se tiene en cuenta.
  • Qué aprendizaje se espera favorecer.

Qué no debe hacer

  • Elegir un problema muy abstracto.
  • Proponer algo que no se entienda en niños pequeños.
  • Arrancar directamente con IA o con código.
La lógica correcta es: primero el aula, después la necesidad, luego la solución.

Paso 2. Definir la solución

Una vez identificada la problemática, se plantea cómo se responderá a ella usando juego, retroalimentación inmediata y apoyo tecnológico.

Qué debe definir

Se debe redactar una solución concreta. Ejemplo: diseñar una experiencia donde el niño muestre o identifique un objeto y el sistema apoye la clasificación para que el niño aprenda de forma visual y participativa.

Elementos mínimos de la solución

  • Qué hará el niño.
  • Qué hará el sistema o modelo.
  • Qué hará el docente.
  • Qué aprendizaje se fortalecerá.

Relación con el juego

La solución debe plantearse como una experiencia lúdica, no como una prueba técnica. El niño participa, observa, recibe respuesta y corrige.

Paso 3. Elegir las categorías

Aquí se decide exactamente qué reconocerá la IA. Se recomienda trabajar solo con dos o máximo tres categorías para que el modelo sea manejable y la actividad sea comprensible.

Cómo elegirlas

Las categorías deben derivarse directamente de la problemática. Si el problema es clasificación de residuos, las categorías pueden ser plástico y papel. Si el problema es emociones, podrían ser feliz y triste. Si el problema es colores, podría trabajarse con rojo y azul.

Criterios

  • Que sean visibles y distinguibles.
  • Que tengan sentido en niños pequeños.
  • Que puedan mostrarse con ejemplos reales.

Relación con abstracción

Este paso muestra el pilar de abstracción, porque se reduce el problema solo a lo esencial para poder trabajar una clasificación simple.

Paso 4. Recolectar los datos

Antes de entrenar el modelo, se deben reunir las imágenes o muestras que alimentarán la herramienta.

Cuántos datos se recomiendan

Para un ejercicio de aula, se recomienda mínimo 10 imágenes por categoría y, si es posible, entre 15 y 20 para lograr un resultado más estable. Las imágenes deben variar en ángulo, iluminación y forma del objeto, sin perder claridad.

Cómo reunirlos

  • Tomar fotos con celular.
  • Buscar imágenes propias y organizarlas en carpetas.
  • Separar cada categoría por carpeta antes de subirlas.

Ejemplo de organización

  • Carpeta 1: plástico.
  • Carpeta 2: papel.
  • Nombrar los archivos o al menos mantenerlos ordenados.
No basta con decir “datos”; aquí deben verse las imágenes o muestras que realmente se usaron para entrenar el modelo.

Paso 5. Entrenamiento del modelo

En este paso se usa Teachable Machine para crear el modelo de reconocimiento.

Ruta básica

  • Entrar a Teachable Machine.
  • Seleccionar Get Started.
  • Elegir Image Project.
  • Seleccionar Standard Image Model.

¿Hay que registrarse?

Para crear y entrenar un modelo básico no siempre es obligatorio iniciar sesión. Sin embargo, si se quiere guardar, compartir o administrar mejor el trabajo, conviene usar una cuenta de Google.

¿Qué significa trabajar desde Drive?

Si el usuario está con cuenta Google, puede apoyarse en Drive para guardar materiales, organizar carpetas o conservar evidencias. No es obligatorio para entrenar, pero sí útil para orden y respaldo.

Qué debe hacer en la plataforma

Cambiar el nombre de las clases por las categorías elegidas, cargar las imágenes correspondientes en cada una y pulsar entrenar cuando las clases tengan suficientes ejemplos.

Paso 6. Probar el modelo

Después del entrenamiento, se valida si el modelo realmente reconoce lo esperado.

Cómo se prueba

  • Activar la cámara desde la herramienta.
  • Mostrar objetos reales frente a la cámara.
  • Observar la categoría detectada y el porcentaje de confianza.

Qué debe registrar

  • Casos en los que acierta.
  • Casos en los que falla.
  • Nivel de confianza mostrado.

Qué debe concluir

La IA reconoce, pero todavía no decide qué hacer. Esta prueba sirve para ver si el reconocimiento es suficientemente claro para pasar luego a la lógica.

Paso 7. Aplicar los pilares del pensamiento computacional

En este momento ya se debe mostrar explícitamente cómo intervienen los cuatro pilares.

Descomposición

Dividir el problema en partes: reconocer objeto, identificar categoría, decidir acción y dar respuesta.

Reconocimiento de patrones

Observar qué rasgos se repiten en las imágenes o materiales de cada categoría.

Abstracción

Quitar detalles innecesarios y dejar solo la característica clave para la clasificación.

Algoritmos

Definir instrucciones ordenadas a partir del resultado de la IA.

Aquí no basta con nombrar los pilares; deben mostrarse aplicados al proyecto elegido.

Paso 8. Construir la lógica

El resultado de la IA se convierte en una entrada para la toma de decisiones.

Cómo se hace

Se toma el objeto detectado y el porcentaje de confianza. A partir de eso se formulan reglas. Ejemplo: si detecta plástico y la confianza es mayor a 80, indicar contenedor azul; si detecta papel y la confianza es mayor a 80, indicar contenedor verde; si la confianza es baja, pedir repetir.

Qué debe verse

  • Reglas escritas con claridad.
  • Condiciones simples.
  • Relación entre reconocer y actuar.

Cómo se puede presentar

Puede presentarse como lista de reglas, tabla de decisiones o frases condicionales del tipo “si... entonces...”.

Paso 9. Hacer el diagrama de flujo

El diagrama organiza visualmente la lógica y permite mostrar el proceso de principio a fin.

Estructura básica

  • Inicio.
  • Entrada del dato o lectura del objeto.
  • Decisión sobre confianza.
  • Decisión sobre categoría.
  • Acción o salida.
  • Fin.

Qué debe aclarar

El diagrama no es la IA. El diagrama representa la lógica que responde a lo detectado por la IA.

Cómo presentarlo

Puede hacerse a mano con buen orden, en digital, o en tamaño ampliado para la exposición del mural.

Paso 10. Llevarlo a PSeInt

En PSeInt se simula la secuencia de decisiones para mostrar que la lógica también puede expresarse como pseudocódigo.

Qué debe hacer

Tomar el diagrama y convertirlo en instrucciones. Se leen variables como objeto y confianza, y luego se usan condicionales para decidir la acción.

Qué debe mostrarse

  • Código claro y ordenado.
  • Pruebas con datos de ejemplo.
  • Salida o respuesta del sistema.

Qué debe aclararse

No se está conectando PSeInt con la IA real; se está simulando la lógica para evidenciar el proceso de pensamiento computacional.

Paso 11. Diseñar la actividad infantil

El proyecto debe aterrizarse a una experiencia para niños pequeños, no quedarse en herramienta y código.

Qué debe explicar

Cómo participa el niño, qué observa, qué hace, qué recibe y qué aprende. Ejemplo: el niño toma un objeto, lo muestra, escucha o ve la respuesta y corrige la clasificación.

Qué elementos lúdicos puede incluir

  • Juego de clasificación.
  • Colores visibles para cada acción.
  • Retroalimentación inmediata.
  • Participación guiada por el docente.

Qué aprendizaje debe evidenciar

Formación de hábito, comprensión simple, participación activa y uso de tecnología como apoyo al aprendizaje significativo.

Paso 12. Construcción del mural

El mural es la evidencia visual y expositiva del proyecto. Debe mostrar el proceso completo, no solo decorar.

Cómo organizarlo

Se recomienda dividirlo en cuatro secciones grandes y visibles: problemática y solución, datos del entrenamiento, lógica y diagrama, y simulación o demostración final.

Sección 1. Problema y solución

  • Problemática redactada.
  • Población o contexto.
  • Solución propuesta.
  • Explicación breve del propósito pedagógico.

Sección 2. Datos

  • Ejemplos de imágenes usadas.
  • Cantidad por categoría.
  • Cómo se recolectaron.
  • Qué categorías se definieron.

Sección 3. Lógica y diagrama

  • Reglas de decisión.
  • Condiciones.
  • Diagrama de flujo ampliado.

Sección 4. Código y prueba

  • Fragmento de PSeInt.
  • Capturas de prueba del modelo.
  • Resultado esperado.
  • Aplicación en el aula infantil.

Qué debe evitar

No llenar el mural de texto innecesario ni dejarlo solo en imágenes sueltas. Debe mostrar qué problema había, cómo se pensó la solución, cómo se entrenó el modelo y cómo esa solución se convierte en una experiencia de aprendizaje.
